Введение

Создание собственного шаблона начинается с создания папки в themes. Название папки (например, test) автоматически становится именем шаблона в настройках админки; дополнительно прописывать ничего не нужно.

Структура папок

assets/
  • img — изображения
  • css — стили
  • js — скрипты
views/layouts/
  • main.php — основной layout
  • main-public.php — публичный layout

main.php

Основной layout, отвечающий за общую структуру сайта: <html>, <head>, шапку, футер, подключение CSS/JS и вывод контента страниц.

PHP (Yii2)
<?php
$this->beginPage();
$this->head();
$this->beginBody();
    /* Шапка, контент, футер */
    echo $content;
$this->endBody();
$this->endPage();
  • $this->title — заголовок страницы
  • $content — содержимое конкретного view-файла
  • Url::base() и Url::to() — безопасная генерация URL
  • Yii::$app->user->isGuest — условная логика авторизации

main-public.php

Layout для публичных страниц (главная, инфо, загрузки) без логики личного кабинета.

PHP (Yii2)
<body class="layout-public" data-route="...">
  • Класс layout-public позволяет применять разные стили и JS-логику
  • Атрибут data-route помогает определять текущий маршрут на фронте

CSS-конвенции

Auth / Public
  • .auth-form — контейнер форм авторизации
  • .login-form — форма входа
  • .form-title — заголовок формы
  • .captcha-inline — горизонтальное расположение captcha
Backend
  • .backend-default-index — главная админки
  • .login-box — центрированный контейнер формы входа
  • .box — общий контейнер блока
  • .box-primary — основной блок с акцентом
Shop / Game Servers
  • .game-servers-index — список серверов
  • .shop-category — контейнер категории магазина
  • .pack-items — список предметов внутри пака
  • .item-form — форма предмета
Bonuses
  • .bonuses-index — страница списка бонусов
  • .bonuses-form — форма создания/редактирования
  • .bonus-codes-index — список бонус-кодов

Кнопки

Используйте .btn-* для обозначения действий: primary — основное, success — создание, danger — удаление, warning — статус.

Совет: сохраняйте файл в \protected\l2j и следуйте конвенциям именования классов — это упростит поддержку и расширение шаблона в будущем.
